<p>After building the model, we will want to test its performance against new data. It's important that this data has not been seen before during the model training. To achieve this we have to reserve part of our dataset for testing, which will be removed from the training phase.</p>
<p>We'll reserve 20% of the total dataset for testing. The random_state variable is for initializng the randomizer. By hardcoding it here we make sure that we select the same records everytime that we run the script.</p>

<p>Now we have 4 variables:</p>
<ul>
<li><code>X_train</code>: The input features of the training dataset.</li>
<li><code>X_test</code>: The input features of the test dataset.</li>
<li><code>y_train</code>: The target of the training dataset.</li>
<li><code>y_train</code>: The target of the test dataset.</li>

<p>How can we express the accuracy of the model in a more mathematical way? For that we use a quality metric, in this case we could use <a href="https://en.wikipedia.org/wiki/Root-mean-square_deviation" rel="nofollow">RMSE</a>.</p>
